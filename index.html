SYSTEM OVERVIEW · 总览

01系统架构：智能在 GPU，控制权在本体

VLA / VLM 等重模型跑在局域网 GPU 服务器，本体只做控制与执行——分层为了算力，红线为了安全： 智能与实时控制解耦，所有运动指令由 Robot 本地仲裁与执行，这是全页所有设计的第一原则。

安全边界：GPU 侧代码无任何 ROS 发布能力（仓库级保证）；ACT action chunk 必经 Robot 本地 Streaming Executor——输入看门狗 0.5s、joint-state 陈旧即断、controller 模式周期校验；ACT 仅在 MoveIt pregrasp 实测到位后，从实测关节上下文启动。

02抓取体系：标定 · 传统 · 学习 核心工作

围绕「抓得住、抓得稳」的完整技术栈：毫米级标定打底，确定性归传统方法， 接触丰富的最后一段交给 ACT——三层各做自己擅长的事。

2.1标定体系：视觉引导抓取的精度地基

四层标定 + 一套误差分解验证法，把末端切平面精度从开环 8.6mm 压到补偿后 1.1–5.8mm、闭环 2–6mm。

① 关节零位标定

FK 与安全的基线：按组释放扭矩手摆 URDF 零位记录 offset，逐关节走行程记录 raw_min/max， 行程转 MoveIt 软限位。人眼对零约 1° 级——够用，且被上层外部真值反查兜底。

产物：config/*_calibration.yaml → xacro offset / 行程限位

② 头相机手眼（眼在手上 AX=XB）

16-tag 固定板 + 头 pan/tilt 多姿态，全板角点 bundle PnP，重投影 >1px 拒帧，双向逼近吸收回差。

平移 RMSE 1.26mm · 旋转 0.16° · 三轮重复 std ≤1mm → 写入 URDF

③ 右臂手眼（眼在手外 AX=YB）

头固定当裁判，夹爪 tag 随臂动；相机位与 tag 安装位双未知联解（Shah + LM 精修）。 在目标工作空间（桌面 z≈0.70–0.85m）采样——在哪儿干活，就在哪儿标定。

22 样本 RMS ~6mm → tag 安装位姿 + 臂底座修正 C 写入 URDF

④ 补偿与视觉闭环

常值补偿 (+7.5, +0.1, +3.7)mm 持久化于配置文件（含有效范围）；视觉闭环同帧相对测量， 手眼 / FK 系统误差自然抵消，切平面只修 1–2 轮。

开环切平面 1.1–5.8mm → 闭环 2–6mm（目标 ≤1cm 达成）
验证方法 · CALIB 残差法（自建）：视觉链与 FK 链对同一物理点独立解算取差， 把标定误差从规划 / 执行 / 检测误差中分离——同点重复 ≤1.5mm 证明误差是系统性、可补偿的； 误差分解：标定链 3D 均值 8.6mm · 执行环节 dz −21mm（重力下垂）· 相机噪声底 0.3mm。 残差反查关节零偏：wrist_flex 零位偏 +9.19°（拟合代入后 RMS 6.2→3.5mm）——标定顺手当了回体检。
决策与权衡：
  • 低 RMSE ≠ 对：12-tag 时 RMSE 小但板轴方向错（平面 PnP 歧义），必须 16 tag 全可见才采信；
  • 模型选错，数据再好也没用：旧版误用「固定板」模型解 tag-on-gripper 数据，得出虚假 102mm 误差；
  • 达标即止：零偏 9.19° 诊断后未写回——末端指标已达 ≤1cm，精修路径论证存档，不无限追物理真值；
  • 修正只打臂底座 C，绝不动头相机外参：保住视觉链的独立真值锚点。

2.2传统方案：分割 + 抓取网络

确定性归传统：可解释、可复现、可回归，每一步都有明确的成功判据与错误码。

管线：指定物体 → VLM + SAM2 分割掩码 → GPD / PointNetGPD 抓取位姿候选 （~1s @ D455 场景，GPU HTTP 服务）→ 姿态约束排序（顶抓 / 侧抓）→ MoveIt 规划 → 本地校验执行。

分割质量决定候选质量

VLM bbox 粗框 + SAM2 精确掩码（多种 mask 模式实测对比），把目标点云从桌面场景中干净分离， 是抓取网络出好候选的前提。

抓取网络双后端实践

GPD（atenpas/gpd C++）采样候选 + PointNetGPD 三分类模型重评分，两个后端同场景对比实测， 按成功率与时延取舍。

失败可解释

检测、分割、候选、规划、执行每步独立判定——空抓、桌沿误检等失败能定位到具体环节并回归， 不依赖"再试一次看看"。

2.3两阶段 ACT：确定性粗接近 + 学习闭环精抓

ACT 不学挥臂搜索（交给传统层），只学接触丰富的最后 5–15cm——示教范围故意做窄，数据效率与成功率双高。

阶段一：MoveIt pregrasp 粗接近

确定性、安全限位、可回归
  • IK 种子序列 → MoveIt 规划 → 本地碰撞 / 可达校验
  • 单点 4s 轨迹执行，实测关节到位才放行
  • 把臂安全合法地送到「门口」，不为此录几十条示教

阶段二：ACT 闭环精抓

桌面接触、物体滑移、位姿微调——解析建模性价比最低的部分
  • ACT（ResNet18，chunk 100）从实测 pregrasp 上下文启动
  • 推理起点与训练采集分布严格对齐，避免分布外初始状态
  • action chunk 经本地 Streaming Executor 执行（0.5s 看门狗）
  • VerifyGrasp 腕部复核 + 夹爪信号双判定，失败自动重抓
数据闭环：30Hz 双相机采集（.incomplete 校验后原子发布） → rsync 断点续传 → GPU 重校验转 LeRobot → 训练（ResNet18 · chunk 100 · batch 8 · 5000 steps，可断点恢复） → 强制校验六维 state/action 后版本化导出 → 资产目录激活部署，可回滚。 关键工程细节：用 predict_action_chunk 每请求取完整 action chunk—— select_action 的内部队列会吞掉随后 99 步的新观测，直接破坏闭环。

PILLAR 02 · 走得准 · 到得了

03移动基础：建图与导航

为抓取和送达提供可信的移动基础——标定先行、异常可防、目标业务化、脱困有退路。

里程计物理标定（根因治理）

  • 1m 直线标轮半径、360° 原地旋转标轮距（验收 5–10%）
  • 固定「传感器 → TF → 里程计 → SLAM」排查顺序 + 症状速查表，不盲调 SLAM 参数
  • 雷达本体遮挡 mask 校准，屏蔽给 inf 而非假障碍
根因故事：轮距配置 0.25m vs 实际 0.49m → yaw 积分错 2 倍 → 转弯重影的真凶

slam_mapping_guard 异常防护（自研）

guard 把 scan 投影到当前地图做穿墙检测，三级输出而非简单阈值熔断： SUSPECT 只提示不锁车；CRITICAL（多束激光连续穿可信墙体）停发 scan + 暂停建图 + 软锁底盘， 阻止错误 free space 写进 pose graph。恢复走状态机：UNLOCK → 定位确认 → RESUME。

架构：lidar → /scan_raw → guard → /scan → SLAM

导航到桌面业务链

  • 桌位 / 送达点标注为命名地点，业务语义直接生成导航目标
  • 三阶段对接：Nav2 到桌旁 → Spin 收敛朝向 → 自研精确 dock 贴桌（含横向误差校正）
  • 定位质量门控：AMCL 协方差 + scan-map 一致性双条件，不达标自动重定位——坏定位不进任务
NavigateToNamedPlace / AutoLocalize

脱困与恢复

  • 卡住检测（15s 位移 < 0.1m）→ 恢复行为编排：清 costmap / 原地转 / 倒退，最多 6 轮
  • 差异化 inflation_radius：解决「门被膨胀堵死」
  • RotationShim + RPP 替换 MPPI：解决 CPU 卡顿导致的控制循环 miss
全部结论来自真机现象定位，非参数盲调
决策与权衡 · 为什么不自动回滚地图：快照 / 自动回滚只立架构策略、不在当前版本实施—— 自动回滚必须等定位 gate 足够可靠后再接入，否则可能把错误 pose graph 回滚成另一个不可解释状态。 异常处理的价值不在于「出事后自动动作」，而在于让每一步都可人工确认、可解释。

4.3交付工程化：以验收标准倒推设计

离线安装、原子升级、可回滚、可卸载、可远程诊断。

离线 Release + 升级回滚

SHA-256 manifest 校验；.staging 解包 + current 原子切换；重复安装幂等；升级后 doctor 门禁，失败自动回退；客户机零构建。

xlerobotctl install-release / rollback
双模式卸载

--keep-assets 保留配置 / 资产 / 日志；--purge 对服务、规则、用户、目录逐项确认无残留。

uninstall.sh --keep-assets | --purge
诊断与日志纪律

doctor 结构化体检；support-bundle 脱敏打包支撑远程定位；持久日志统一 /var/log/xlerobot + logrotate；任务级 JSONL trace 带唯一 task_run_id。

xlerobotctl doctor / support-bundle
四 Profile 互斥

demo / mapping / calibration / collection 固定 launch 组合，systemd Conflicts + flock 双重互斥，同一时刻只有一个 profile 持有硬件。

xlerobotctl start demo

4.4工程方法：我怎么工作

真机优先

所有设计决策以真机回归为准。git 历史是一条「现象 → 定位 → 修复 → 回归」的实证链：空抓、Spin 后横向偏差、导航确认竞态，都是在真机前发现并修复的。

契约与测试

接口先行（39 个 action / srv / msg 契约，零死定义），契约测试锁定已验证行为，验收标准可机器检查。

AI 协作工程化

逐行实现大量借助 AI；我负责需求分解、架构决策、代码评审与真机验证。AI 产出的每一行都要过评审与回归——这是我对 AI 时代工程师工作方式的践行。

评审文化

交付前完成四领域并行审查 + 关键结论独立复核，输出带 path:line 证据链的审查报告，并按 P0/P1 跟踪修复。
